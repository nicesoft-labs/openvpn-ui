Действия перезапуска

Перезапуск сервера   Условный перезапуск сервера OpenVPN применит новую конфигурацию и повторно инициирует все активные клиентские сессии.
Перезапуск контейнера   Полный перезапуск контейнера OpenVPN для повторного применения правил брандмауэра и перезапуска сервера OpenVPN. Занимает больше времени, чем условный перезапуск.
Перезапуск UI   Перезапуск контейнера OpenVPN-UI. Текущая веб-сессия будет отключена.

Действия удаления

ОПАСНАЯ ЗОНА! Вы не сможете отменить действия из этого раздела!
Удалить PKI   Удалить PKI, все серверные ключи OpenVPN и клиентские сертификаты. Это предотвратит подключение клиентов с текущими сертификатами!
Удалить .ovpn   Удалить файлы "*.ovpn" из каталога "/etc/openvpn/clients".
Удалить статические   Удалить все статические IP-резервации из каталога "/etc/openvpn/staticclients".
УДАЛИТЬ ВСЁ   Эта кнопка выполнит все шаги выше по очереди — удалит PKI, серверные ключи, клиентские сертификаты и файлы "*.ovpn".
Удалить БД   Удалить базу данных OpenVPN-UI. После удаления перезапустите OpenVPN-UI. База данных будет создана заново при следующем запуске контейнера OpenVPN-UI.

Действия инициализации

ОПАСНАЯ ЗОНА! Вы не сможете отменить действия из этого раздела!
Будьте осторожны, необходимы знания! Краткое руководство EasyRSA, Подробная документация EasyRSA.
Памятка по инициализации
Выполните действия ниже последовательно:
  1. Скопируйте переменные EasyRSA (кнопка EasyRSA VARs).
  2. Сгенерируйте новый список отозванных сертификатов (CRL).
  3. Инициализируйте PKI.
  4. Создайте центр сертификации (CA).
  5. Сгенерируйте серверный сертификат и подпишите запрос.
  6. Сгенерируйте ключ Диффи-Хеллмана.
  7. Сгенерируйте TA-ключ OpenVPN.
  8. Кнопка INITIALIZE ALL выполнит все шаги по порядку автоматически.

  Создать новый серверный сертификат, ключ и файлы шифрования командой "easyrsa gen-req & sign-req server nopass".
Сгенерировать DH

INITIALIZE ALL   Эта кнопка выполнит все шаги выше по порядку — скопирует переменные EasyRSA и создаст PKI, CA, сервер, DH, TA и CRL.
